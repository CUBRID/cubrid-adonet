Known Issues
============
Now, ADO.NET beta driver did not support CUBRID 9.0.0 beta version completely. Please refer the following.

- APIS-366 : When using bind parameters to insert data, if parameter type is SET and the actual data type is STRING, Ado.net driver will threw an exception. 
  When using bind parameters to insert data, if parameter type is CCI_U_TYPE_SET/CCI_U_TYPE_MULTISET/CCI_U_TYPE_SEQUENCE
  and the actual data type is CCI_U_TYPE_CHAR/CCI_U_TYPE_NCHAR/CCI_U_TYPE_STRING/CCI_U_TYPE_VARNCHAR.
  After data sort, the first data length must be greater than or equal to other data. otherwise,
  the Ado.net driver will threw an exception.
  For more information, please refer to the url: http://jira.cubrid.org/browse/APIS-366
